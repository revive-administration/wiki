====== 데이터 테이블 작성 요령 ======

데이터 테이블 작성은 [[https://ko.libreoffice.org/download/libreoffice-stable/|리브레 오피스]]로 작업하는 것을 원칙으로 한다. 엑셀로 작업하는 것도 가능하긴 하지만 추가로 인코딩 변경등의 작업이 필요하기 때문에 전달하는 과정에서 실수로 인코딩 변경이 되어 있지 않을 경우 한글이 깨지는 등의 문제가 발생할 수 있다.

엑셀로 작업하고 싶을 경우 엑셀로 작업을 완료한 후 Notepad++ 등의 에디터에서 인코딩 방식을 Unicode(UTF-16)으로 바꿔주자

===== 1. 작업환경 만들기 =====

데이터 테이블의 확장자는 csv이다. 엑셀을 기본프로그램 으로 해두었다가 실수로 파일을 열람한 뒤 저장을 하면 인코딩 환경이 바뀌는 참사의 가능성이 있다. 일반적인 컴퓨터 사용환경에서 csv파일을 다루는 경우는 흔하지 않기때문에 어차피 csv 파일 쓸일은 없다고 생각해도 무방하니 csv확장자에 대한 연결 프로그램 설정을 리브레 오피스로 해두자.

===== 2. 작업 요령 =====
데이터 파일의 저장/열람/생성시의 인코딩 방식은 항상 **UTF-16**이다. 처음 문서를 만든다면 Unicode(UTF-16)을 선택한 후 작업을 해주고, 수정 작업일 경우 파일을 열때 Unicode(UTF-16)으로 열어주면 된다. 


==== 주의사항 ====
  * 새로 데이터를 만들경우 구글드라이브에 있는 샘플 문서를 참고하여 작업하는걸 기본으로 하자. 컬럼 네임은 기존 데이터에서 복사해서 붙여주면 완벽. 저장시의 확장자는 항상 csv이다.
  * 컬럼 값에 해당하는 내용이 빈칸이더라도(예: 파티클 정보 등의 에디터 상에서 넣어주는 요소들) 컬럼을 명시해주자. 컬럼 명시를 해주지 않으면 임포트시에 해당 컬럼이 없다는 에러가 뜬다. 
  * 간혹 복붙할때 저장할 수 없는 서식 또는 내용이 포함되어 있다는 문구의 팝업이 뜨는데 **텍스트 CSV 형식 사용** 버튼을 눌러주면 된다.
  * VE_로 시작하는 Enum값들의 디폴트 값은 보통 VE_None이다. (물론 아닐수도 있음)
  * 굳이 작성 안해도 데이터(예:VE_None)는 공란으로 두어도 무관하다. 

===== 3. 데이터파일 네이밍 =====
^ 구분 ^ 네이밍 예시 ^ 인코딩 ^ 설명 ^
|AutoSkill|2016-12-16/DT_AutoSkill.csv|Unicode(UTF-16)|2016년 12월 16일 첫번째 수정작업 파일|
|AutoSkill|2016-12-16-3/DT_AutoSkill.csv | Unicode(UTF-16)|2016월 12월 16일 3번째 수정작업 파일|

언리얼에서는 데이터 임포트시에 데이터의 이름에 관계 없이 임포트가 가능하기 때문에 날짜와 횟수 첨자로 파일의 최신여부 구분을 하도록 하자.

위의 예시처럼 앞에 DT_ prefix를 넣어준 뒤 폴더로 날짜로 구분한다. 만약 같은날 동일 파일 작업을 2회 이상 하였다면 횟수 첨자를 추가해주자. 횟수 첨자가 없는 파일은 해당 날짜 첫번째 작업 파일이다.

===== 4. 인코딩에 관하여 =====
언리얼의 디폴트 인코딩은 UCS-2 LE BOM 이라는 인코딩 방식인데 다른 말로 UTF-16라고도 한다(우왕!). 보통의 다국어 문서 작업에서는 UTF-8을 많이 사용하고 언리얼 역시 UTF-8를 지원해주지만 작업의 편의를 위하여 디폴트를 따라가도록 하자. 혹시 인코딩의 장단점이 궁금하다면 [[https://docs.unrealengine.com/latest/INT/Programming/UnrealArchitecture/StringHandling/CharacterEncoding/#textformats|이 문서]]를 참고하자. 



간혹 문서가 깨지는 경우는 해당 문서가 언리얼 에디터에서 UTF-8 파일로 익스포트(추출)된 경우이다. 익스포트 하면 UTF-8로 나오는 이유는 해당 데이터 테이블이 임포트를 한 적이 있고 그 인코딩 방식이 UTF-8였기 때문이다. 당분간은 해당 문제가 발생할 수 있어 UTF-8로 열어주거나 UTF-8로 해도 깨질경우 엑셀 혹은 Notepad++에서 열어준 뒤 수정 후 저장을 UTF-16(UCS-2 리틀 엔디언)으로 꼭 해주자.
